# Doc Writer - 官方文档写作 Skill

> 本 Skill 仅在代码完全跑通、进入文档更新阶段时激活。

## 写作原则

1. **代码即文档**：文档必须与当前代码完全一致，不允许描述已废弃的接口
2. **面向读者**：假设读者是刚接手项目的嵌入式开发工程师，具备 C 语言基础但不了解本项目
3. **精简优先**：每段不超过 5 行，能用表格代替文字时优先用表格

## README 更新规范

当更新 README.md 时，必须包含以下章节：

```markdown
# 项目名称

## 概述
一句话说明项目做什么、面向什么硬件平台。

## 硬件要求
| 项目 | 型号/规格 |
|------|----------|
| 芯片 | HiSilicon 3863 |
| 开发板 | 小熊派 / BearPi |
| ... | ... |

## 快速开始
1. 克隆仓库
2. 编译命令
3. 烧录命令
4. 预期输出

## 目录结构
简述关键目录的用途。

## API 接口
列出对外暴露的核心函数，附简要说明。

## 已知问题
列出当前已知的限制或 Bug。
```

## 接口文档规范

当为函数编写接口文档时，使用以下格式：

```c
/**
 * @brief  一句话功能描述
 * @param  param_name  参数说明
 * @return 返回值说明
 * @note   使用注意事项（如有）
 */
```

## C/C++ 注释规范

- 函数头注释：使用 `/** */` Doxygen 格式
- 行内注释：使用 `//`，只在逻辑不直观时添加
- 禁止注释掉的代码残留（用 git 历史代替）

## 卸载条件

文档生成完毕并确认与代码一致后，本 Skill 自动关闭。
