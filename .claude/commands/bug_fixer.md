# Bug Fixer - 深度自审 Skill (Reflexion)

> 本 Skill 仅在 git commit 前或排查死机 Bug 时激活。激活后执行以下 Reflexion 反思链。

## Reflexion 反思链

当遇到难以定位的 Bug 时，必须按以下步骤执行深度自审：

### 第 1 轮：现象收集
1. 完整阅读错误日志/崩溃信息，逐行标注关键线索
2. 确定 Bug 的触发条件：必现 vs 偶现、特定输入 vs 随机
3. 记录 Bug 的影响范围：哪个模块、哪个函数、哪个时间段引入

### 第 2 轮：假设生成
针对以下 5 个常见嵌入式 Bug 根因，逐一生成假设：

| 编号 | 根因类别 | 检查要点 |
|------|---------|---------|
| 1 | 内存问题 | 越界写、use-after-free、double-free、栈溢出、堆碎片 |
| 2 | 并发问题 | 竞态条件、死锁、优先级反转、中断与主循环共享变量未加保护 |
| 3 | 硬件交互 | 寄存器时序错误、DMA 缓冲区对齐、外设初始化顺序 |
| 4 | 通信协议 | 帧解析偏移、字节序、校验和计算、超时处理 |
| 5 | 逻辑错误 | 边界条件、类型溢出、隐式转换、未处理的错误返回值 |

### 第 3 轮：对抗性审查
对每个假设执行以下审查：

- **反向论证**：如果这个假设是错的，什么证据能推翻它？
- **代码走读**：从 Bug 现场反向追溯调用链，逐帧检查数据流
- **工具辅助**：建议使用 Valgrind、AddressSanitizer、GDB、逻辑分析仪等工具验证

### 第 4 轮：Root Cause 输出报告

输出格式：

```
=== Bug Fixer Reflexion Report ===
现象：[简述 Bug 表现]
触发条件：[必现/偶现，触发场景]
根因分析：
  - 假设 1: [描述] → [证据] → [结论：排除/确认]
  - 假设 2: [描述] → [证据] → [结论：排除/确认]
  - ...
Root Cause: [最终确认的根因]
修复方案：[具体代码修改建议]
预防措施：[如何避免同类问题再次发生]
===================================
```

## 交叉审查模式 (git commit 前)

当在 `git commit` 前激活时，执行以下快速审查：

1. `git diff` 逐行审查所有改动
2. 检查是否引入了新的安全隐患（缓冲区溢出、未校验输入）
3. 检查是否破坏了既有接口的兼容性
4. 输出审查结论：PASS / FAIL（附理由）

## 卸载条件

找出 Root Cause 并输出报告后，或 git commit 审查完成后，本 Skill 自动关闭。
