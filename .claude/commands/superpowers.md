# Superpowers - 纪律执行官

> 本 Skill 仅在正式编写或重构 C/C++ 核心代码时激活。激活后强制执行以下流程。

## 强制 TDD 流程

在编写任何 C/C++ 功能代码前，必须严格遵循以下顺序：

1. **Red - 先写失败的测试**
   - 在编写功能代码之前，先编写一个会失败的测试用例
   - 测试必须覆盖：正常路径、边界条件、错误输入
   - 对于嵌入式代码，测试应考虑：中断上下文、DMA 传输边界、内存对齐

2. **Green - 写最少代码让测试通过**
   - 只编写刚好让测试通过的代码
   - 禁止添加"以防万一"的额外功能

3. **Refactor - 重构**
   - 测试通过后，清理代码结构
   - 确保每次重构后测试仍然通过

## 原子化修改 (Atomic Changes)

每一条 commit 必须满足：

- **单一职责**：一次只改一件事。如果需要改三件事，拆成三个 commit。
- **可编译**：每个 commit 都必须能独立编译通过，不允许出现"中间状态"的编译错误。
- **可回滚**：每个 commit 都可以安全 revert 而不会破坏其他功能。

## 嵌入式特殊检查清单

在提交 C/C++ 代码前，必须检查：

- [ ] 所有 `malloc` 是否有对应的 `free`
- [ ] 中断处理函数 (ISR) 中是否避免了阻塞操作
- [ ] 是否检查了指针非空再解引用
- [ ] 栈上大数组是否可能导致栈溢出
- [ ] 外设寄存器操作是否使用了正确的 volatile 修饰
- [ ] DMA 缓冲区是否满足对齐要求

## 卸载条件

代码修改完成并通过初步编译/测试后，本 Skill 自动关闭，恢复正常工作模式。
