# 智能健康监测系统 - 项目展望与开发方向

## 一、项目背景

随着现代社会生活节奏的不断加快，人们对自身健康状况的关注程度逐渐提高，尤其是健康问题已经成为影响大众生活质量的重要因素之一。

传统医疗监测设备通常体积较大、操作复杂，并且难以实现实时的数据传输与智能分析，无法满足日常生活中的便携式健康监测需求。

基于此，本项目以**星闪（NearLink）无线通信技术**为核心，结合**嵌入式系统**、**传感器技术**以及**人工智能分析方法**，设计并实现一套**智能手指心跳检测系统**。

系统通过实时采集人体指尖血流信号，完成心率数据监测，并通过无线通信技术将数据传输至上位机进行分析与展示，从而为用户提供更加直观和智能的健康监测体验。

## 二、现有系统基础

### 2.1 硬件平台

系统核心硬件平台为 **Hi3863（WS63）星闪开发板**，具备以下特点：
- 强大的嵌入式处理能力
- 原生支持 NearLink SLE 协议
- 低功耗、低时延的数据传输

### 2.2 已完成的传感器驱动

| 传感器 | 接口 | 功能 | 状态 |
|--------|------|------|------|
| MAX30102 | I2C (GPIO7/GPIO8) | 心率血氧检测 | ✅ 已完成 |
| MPU6050 | I2C (GPIO7/GPIO8) | 六轴运动检测 | ✅ 已完成 |
| KY-039 | ADC (GPIO0) | 心率波形采集 | ✅ 已完成 |

| 模块 | 功能 | 状态 |
|------|------|------|

| RGB三色灯 | 状态指示 | ✅ 已完成 |

### 2.4 通信系统

系统利用 **NearLink 星闪低功耗通信技术（SLE）** 构建无线通信链路：
- **Server端**：采集传感器数据并发送
- **Client端**：接收数据并通过串口转发至PC/手机

## 八、数据格式定义

### 8.1 传感器数据格式

#### MAX30102 心率血氧
```c
uint32_t heart_rate;    // 心率 (BPM), 范围: 40-200
uint32_t spo2;          // 血氧 (%), 范围: 70-100
```

#### MPU6050 六轴数据
```c
float accel[3];         // 加速度 [x,y,z] (g)
float gyro[3];          // 陀螺仪 [x,y,z] (°/s)
float temperature;      // 温度 (℃)
```

#### KY-039 心率
```c
uint16_t adc_value;     // ADC值 (0-4095)
uint32_t heart_rate;    // 心率 (BPM)
```

#### SW-420 震动
```c
bool detected;          // 是否检测到震动
uint32_t count;         // 震动次数
```

### 8.2 通信数据格式

#### JSON输出格式
```json
{
  "type": "data",
  "hr": 72,
  "spo2": 98,
  "temp": 36.5,
  "accel": [0.10, 0.05, 1.00],
  "gyro": [0.10, 0.05, 0.02],
  "fall_conf": 0,
  "status": "Status: Normal"
}
```

#### 命令格式（手机端发送）
- `breath_start` - 启动呼吸引导
- `breath_stop` - 停止呼吸引导
- `mute` - 静音
- `confirm` - 确认报警

## 十、总结

本学期的综合实训将在已有项目基础上，围绕**功能扩展**、**系统优化**以及**通信完善**三个方面展开。

### 已完成的核心功能
1. **传感器数据采集** - MAX30102心率血氧、MPU6050六轴IMU、KY-039心率、SW-420震动
2. **跌倒检测系统** - 融合多传感器数据，输出置信度
3. **健康提醒系统** - 多模式提醒（RGB灯、蜂鸣器、震动马达、OLED）
4. **呼吸引导功能** - 4-7-8呼吸法，帮助用户放松
5. **星闪SLE通信** - 无线数据传输，支持双向通信

### 项目特点
- **硬件完整**：8个传感器/设备，覆盖多种健康监测场景
- **算法完善**：跌倒检测、心率分析、呼吸引导等核心算法
- **通信可靠**：星闪SLE低功耗无线通信
- **扩展性强**：预留移动端APP和AI分析接口

通过本项目的开发实践，团队成员能够进一步加深对**嵌入式系统开发**、**无线通信技术**的理解，提升系统设计与工程实践能力，为后续学习与研究奠定良好的基础。
